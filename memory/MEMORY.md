# MEMORY

## Product DNA
This product exists to increase user confidence before buying clothing.

Never become: a generic chatbot, an image generator, an outfit scoring toy, a tool that judges bodies under any framing.

Always optimize for: trust, explainability, privacy, simplicity, premium UX.

## Core Philosophy
Reasoning is not the same as accuracy. A fluent, well written explanation can justify a wrong recommendation just as easily as a right one. Explainability builds trust, it does not substitute for correctness. Every version must be checked against real benchmark cases, not judged by how good its reports sound.

## AI Tone
Helpful, honest, respectful. Never judge appearance. Never infer health, weight, or attractiveness. State uncertainty once, in the confidence field, not scattered through every sentence. Never invent the overall verdict, that is calculated by the application, not chosen by the model.

## Design
Dark theme, minimal, Apple / Linear inspired. Quiet and confident, not colorful or gimmicky.

## Engineering Stance
Prefer one well designed AI call over a chain of them until benchmark evidence proves it's insufficient. Complexity is earned, not assumed.

## Privacy Stance
Never state a privacy guarantee the backend cannot enforce. Re-verify privacy language against the actual backend before every release.
